# Ответ на возражение

## Предварительные проверки

1. Проверь наличие `профиль/стиль.md`
2. Если файла нет — ответь:
   ```
   Для обработки возражений нужен контекст бренда.
   Запусти /start [название-бизнеса] — это займёт 5-10 минут.
   ```
   И **останови выполнение**.

## Сбор контекста

1. Прочитай `профиль/стиль.md` — тон, ценности, запретные темы
2. Прочитай `профиль/бизнес.md` — продукт, ценность, средний чек
3. Проверь `анализ/персоны/` — если есть, определи кто мог бы так возразить
4. Проверь `правила/` — прочитай ВСЕ правила, где `applies-to` содержит "возражение" или "всё"
5. Проверь `решения/` — если есть решения про продажи или коммуникацию, учти

## Отслеживание неопределённости

### Критические разделы (70% веса) — 3 пункта:
1. Суть возражения (что именно говорит клиент)
2. Тип клиента (потенциальный / текущий)
3. Канал (переписка, звонок, комментарий)

### Вспомогательные разделы (30% веса) — 2 пункта:
1. Контекст (что было до возражения)
2. Как обычно отвечает (текущий подход)

### Расчёт

```
Неопределённость = (незаполненных_крит / 3) × 0.7 + (незаполненных_вспом / 2) × 0.3
Цель: ≤ 0.1
```

После каждого ответа пользователя выводи: `[Неопределённость: 0.XX]`

## Уточнение

Если неопределённость > 0.1:

```
Возражение: "$ARGUMENTS"

Пара вопросов:
1. Откуда пришло — переписка, звонок, комментарий?
2. Это потенциальный клиент или текущий?
3. Что ты обычно отвечаешь на такое?
```

Если контекст ясен из аргументов — не задавай лишних вопросов, сразу генерируй.

## Генерация

Создай **2 варианта** ответа на возражение: `$ARGUMENTS`

### Анализ возражения

Сначала покажи краткий разбор:

```
Возражение: [текст]
Тип: [цена | доверие | срочность | конкуренты | непонимание ценности]
Персона: [если определил — имя]
Настоящая причина: [что стоит за возражением]
```

### Варианты ответа

Каждый вариант должен:
- Соответствовать тону из стиль.md
- **Соблюдать все применимые правила**
- Не спорить с клиентом — присоединиться, потом раскрыть
- Быть конкретным (факты, цифры, примеры из бизнес.md)
- Содержать следующий шаг (не оставлять разговор в воздухе)

```
## Вариант 1: [Мягкий / через понимание]

[текст ответа]

## Вариант 2: [Прямой / через факты]

[текст ответа]
```

После вариантов:

```
Какой ближе? Могу доработать или сохранить.
```

## Сохранение

Когда пользователь выбрал:

1. Создай директорию `контент/возражения/` (если нет)
2. Сохрани файл: `контент/возражения/возражение-краткая-тема.md`
3. Добавь метаданные и трассировку:

```yaml
---
type: возражение
created: YYYY-MM-DD
objection: $ARGUMENTS
sources:
  - профиль/стиль.md
  - профиль/бизнес.md
---
```

```markdown
<!-- Трассировка -->
Основано на:
- Стиль: профиль/стиль.md
- Продукт: профиль/бизнес.md
- Персона: [если есть — ссылка]
- Правила: [список применённых правил]
```

4. Подтверди:
```
Сохранено: контент/возражения/возражение-тема.md

Совет: если это возражение повторяется — создай правило:
/rule ответ на возражение "[тип возражения]"
```
