# Коммерческое предложение

## Предварительные проверки

1. Проверь `профиль/стиль.md`. Если нет — попроси рассказать о бизнесе и запусти `/start`.

## Сбор контекста

1. `профиль/стиль.md` — тон, ценности
2. `профиль/бизнес.md` — продукт, ценность, средний чек
3. `анализ/персоны/` — ближайшая персона к клиенту
4. `правила/` — все правила где `applies-to` содержит "кп", "kp" или "всё"
5. `.claude/rules/` — path-scoped правила для КП
6. `решения/` — про продажи, ценообразование
7. **История:** Glob `контент/кп/**/*.md`, последние 3-5 по mtime — структура, формулировки, цены

## Отслеживание неопределённости

### Критические (70%) — 4:
1. Кто клиент (конкретный человек/компания)
2. Что ему нужно
3. Какую проблему решаем
4. Обращение (на ты / на вы)

### Вспомогательные (30%) — 3:
1. Бюджет / ожидания по цене
2. Сроки / дедлайн
3. Особенности / специфика

```
Неопределённость = (незаполненных_крит / 4) × 0.7 + (незаполненных_вспом / 3) × 0.3
Цель: ≤ 0.1
```

После каждого ответа: `[Неопределённость: 0.XX]`

## Уточнение

Если неопределённость > 0.1:

```
Готовлю КП для: $ARGUMENTS

Пара вопросов:
1. Что именно нужно клиенту?
2. Особенности — бюджет, сроки, специфика?
3. Общение — на ты или на вы?
```

Если контекст ясен — не спрашивай, генерируй.

## Генерация

Структура КП. **Соблюдай применимые правила.**

### 1. Шапка
- Для: `$ARGUMENTS`
- От: из бизнес.md
- Дата

### 2. Понимание задачи
2-3 предложения боли клиента его языком.

### 3. Решение
Что предлагаешь и почему это решает. Конкретно.

### 4. Два тарифа

```
|            | Базовый | Расширенный |
|------------|---------|-------------|
| Что входит | ...     | ...         |
| Сроки      | ...     | ...         |
| Результат  | ...     | ...         |
| Стоимость  | ...     | ...         |
```

Цены из бизнес.md или истории КП. Не выдумывай.

### 5. Почему мы
2-3 конкретных факта из бизнес.md и стиль.md.

### 6. Следующий шаг
CTA.

### 7. Контакты
Из бизнес.md.

## После генерации

Показываешь КП и предлагаешь:
```
• Сохранить → положу в черновики
• "короче"/"дороже"/"замени X" → правлю
• "сделай PDF" → оформлю через /format
```

## Инлайн-итерация

"короче", "добавь гарантию", "замени цены на диапазон" — правь последнее КП, не создавай новое.

## Сохранение

1. Если `контент/кп/черновики/` нет — предложи: "Разложу по черновики/отправлено/принято?"
2. Имя: `кп-название-клиента.md` (транслит кириллицы)
3. Фронтматтер:

```yaml
---
type: кп
created: YYYY-MM-DD
client: $ARGUMENTS
status: черновик
sources:
  - профиль/стиль.md
  - профиль/бизнес.md
---
```

4. Трассировка:
```markdown
<!-- Трассировка -->
Основано на:
- Стиль: профиль/стиль.md
- Продукт: профиль/бизнес.md
- Персона: [если применялась]
- Правила: [применённые]
- История КП: [N прошлых]
```

5. Подтверждение:
```
Сохранено: контент/кп/черновики/кп-название.md

Отправишь клиенту — скажи "отправил", перенесу в отправлено/.
Клиент согласился — "принял", перенесу в принято/.

Оформить PDF?
```

## Лайфцикл

- "отправил [клиент]" → `черновики/` → `отправлено/`, `status: отправлено`
- "принял [клиент]" → `отправлено/` → `принято/`, `status: принято`
- "отказал [клиент]" → `отправлено/` → `принято/`? Нет, создай `отказано/` если ещё нет, с `status: отказано` + поле `reason` (юзер расскажет почему)
