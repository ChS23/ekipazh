# Коммерческое предложение

## Предварительные проверки

1. Проверь наличие `профиль/стиль.md`
2. Если файла нет — ответь:
   ```
   Для создания КП нужен контекст бренда.
   Запусти /start [название-бизнеса] — это займёт 5-10 минут.
   ```
   И **останови выполнение**.

## Сбор контекста

1. Прочитай `профиль/стиль.md` — тон, ценности
2. Прочитай `профиль/бизнес.md` — продукт, ценность, средний чек
3. Проверь `анализ/персоны/` — если есть, определи ближайшую персону к клиенту
4. Проверь `правила/` — прочитай ВСЕ правила, где `applies-to` содержит "кп" или "всё"
5. Проверь `решения/` — если есть решения про продажи, учти

## Отслеживание неопределённости

### Критические разделы (70% веса) — 4 пункта:
1. Кто клиент (конкретный человек/компания)
2. Что ему нужно (какая услуга/продукт)
3. Какую проблему решаем (боль клиента)
4. Обращение (на "ты" / на "вы")

### Вспомогательные разделы (30% веса) — 3 пункта:
1. Бюджет / ценовые ожидания
2. Сроки / дедлайн
3. Особенности / специфика

### Расчёт

```
Неопределённость = (незаполненных_крит / 4) × 0.7 + (незаполненных_вспом / 3) × 0.3
Цель: ≤ 0.1
```

После каждого ответа пользователя выводи: `[Неопределённость: 0.XX]`

## Уточнение

Если неопределённость > 0.1, спроси (если информация не в аргументах):

```
Готовлю КП для: $ARGUMENTS

Пара вопросов, чтобы попасть точно:
1. Что именно нужно клиенту? (какая услуга/продукт)
2. Есть ли особенности? (бюджет, сроки, специфика)
3. Как общаться — на "ты" или на "вы"?
```

Если пользователь отвечает коротко — додумай из контекста бизнес.md, но не выдумывай.
Если контекст ясен из аргументов + профиль — не задавай лишних вопросов, сразу генерируй.

## Генерация КП

Создай КП со следующей структурой. **Соблюдай все применимые правила** из `правила/`.

### 1. Шапка
- Для кого (клиент): `$ARGUMENTS`
- От кого (бизнес): из бизнес.md
- Дата

### 2. Понимание задачи
Кратко, 2-3 предложения — покажи что понимаешь боль клиента. Используй язык клиента, не свой.

### 3. Решение
Что предлагаешь и почему это решает задачу. Конкретно, без воды.

### 4. Два тарифа

```
| | Базовый | Расширенный |
|---|---------|-------------|
| Что входит | ... | ... |
| Сроки | ... | ... |
| Результат | ... | ... |
| Стоимость | ... | ... |
```

Базовый — минимально жизнеспособный вариант.
Расширенный — полное решение с бонусами.

Цены бери из бизнес.md (средний чек) или спроси пользователя.

### 5. Почему мы
2-3 пункта — конкретные факты, не "индивидуальный подход". Из бизнес.md и стиль.md.

### 6. Следующий шаг
Конкретный CTA: что клиент должен сделать, чтобы начать.

### 7. Контакты
Из бизнес.md (если есть).

## Сохранение

После показа КП спроси: "Сохранить? Хочешь поправить что-то?"

Когда пользователь подтвердит:

1. Создай директорию `контент/кп/` (если нет)
2. Сохрани файл: `контент/кп/кп-название-клиента.md`
3. Добавь метаданные и трассировку:

```yaml
---
type: кп
created: YYYY-MM-DD
client: $ARGUMENTS
sources:
  - профиль/стиль.md
  - профиль/бизнес.md
---
```

```markdown
<!-- Трассировка -->
Основано на:
- Стиль: профиль/стиль.md
- Продукт: профиль/бизнес.md
- Персона: [если есть — ссылка]
- Правила: [список применённых правил]
```

5. Подтверди:
```
Сохранено: контент/кп/кп-название.md
```
