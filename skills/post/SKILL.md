# Пост в стиле бренда

## Предварительные проверки

1. Проверь наличие `профиль/стиль.md`
2. Если файла нет — ответь:
   ```
   Для создания поста нужен контекст бренда.
   Запусти /start [название-бизнеса] — это займёт 5-10 минут.
   ```
   И **останови выполнение**.

## Сбор контекста

1. Прочитай `профиль/стиль.md` — тон, ценности, запретные темы
2. Прочитай `профиль/бизнес.md` — продукт, клиент, ценность
3. Проверь `анализ/персоны/` — если есть, прочитай персоны
4. Проверь `правила/` — прочитай ВСЕ правила, где `applies-to` содержит "пост" или "всё"
5. Проверь `решения/` — если есть решения про тон или контент, учти

## Генерация

Создай **3 варианта** поста на тему: `$ARGUMENTS`

Каждый вариант должен:
- Соответствовать тону из стиль.md
- **Соблюдать все применимые правила** из `правила/`
- Обращаться к конкретной боли или интересу целевого клиента
- Содержать CTA (призыв к действию)
- Быть готовым к публикации (без плейсхолдеров)
- Быть разным по формату:
  - **Вариант 1:** Информационный / экспертный
  - **Вариант 2:** Сторителлинг / личный
  - **Вариант 3:** Продающий / с оффером

## Формат вывода

Для каждого варианта покажи:

```
## Вариант N: [Формат]

[текст поста]

---
Формат: [информационный | сторителлинг | продающий]
Персона: [если есть — имя из персон]
Правила: [какие правила применены]
Длина: [количество символов]
```

После трёх вариантов спроси:

```
Какой вариант нравится? Могу доработать или сохранить.
Напиши номер варианта или "сохрани все".
```

## Сохранение

Когда пользователь выбрал:

1. Создай директорию `контент/посты/` (если нет)
2. Сохрани файл: `контент/посты/YYYY-MM-DD-краткая-тема.md`
3. Используй сегодняшнюю дату
4. Добавь метаданные и трассировку:

```yaml
---
type: пост
created: YYYY-MM-DD
format: [информационный | сторителлинг | продающий]
topic: $ARGUMENTS
sources:
  - профиль/стиль.md
  - профиль/бизнес.md
---
```

```markdown
<!-- Трассировка -->
Основано на:
- Стиль: профиль/стиль.md
- Бизнес: профиль/бизнес.md
- Персона: [если есть — ссылка]
- Правила: [список применённых правил]
```

5. Подтверди:
```
Сохранено: контент/посты/YYYY-MM-DD-тема.md
```
