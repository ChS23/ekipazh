# Пост в стиле бренда

## Предварительные проверки

1. Проверь наличие `профиль/стиль.md`
2. Если нет — попроси рассказать о бизнесе в свободной форме, а сам запусти `/start` с извлечённым названием. Не останавливайся — веди за руку.

## Сбор контекста

1. `профиль/стиль.md` — тон, ценности, запретные темы
2. `профиль/бизнес.md` — продукт, клиент, ценность
3. `анализ/персоны/` — если есть, возьми основную персону
4. `правила/` — все правила, где `applies-to` содержит "пост", "post" или "всё"
5. `.claude/rules/` — path-scoped правила для постов (если есть)
6. `решения/` — решения про тон/контент
7. **История:** Glob `контент/посты/**/*.md`, последние 5 по mtime — читай их полностью как in-context examples. Извлеки паттерны: длина, зачины, ритм, CTA, эмодзи.

## Проверка повтора тем

Поищи `$ARGUMENTS` или ключевые слова темы в истории (`контент/посты/**`). Если тема уже была за последние 2 недели:

```
Заметил, у тебя уже был пост на похожую тему [DATE]: "[первая строка]".
Это окей — развить тему по-другому или переключимся на что-то другое?
```

Ждёшь ответа прежде чем генерировать.

## Генерация

Создай **3 варианта** поста на тему: `$ARGUMENTS`

Каждый:
- В тоне из `профиль/стиль.md`
- **Соблюдает применимые правила**
- Паттерны из истории постов (длина, ритм, зачины)
- Обращается к конкретной боли персоны
- Содержит CTA
- Готов к публикации, без плейсхолдеров

Три формата:
- **Вариант 1:** Информационный / экспертный
- **Вариант 2:** Сторителлинг / личный
- **Вариант 3:** Продающий / с оффером

## Формат вывода

```
## Вариант N: [Формат]

[текст]

---
Длина: [символов]
Правила: [применённые]
```

После:
```
Какой ближе?
• Номер → сохраню в черновики
• "короче"/"длиннее"/"смешнее"/"убери Х" → правлю выбранный
• "ещё вариант" → новый подход
• "переделай всё" → с нуля
```

## Инлайн-итерация (без перевызова)

Если пользователь отвечает "короче"/"длиннее"/"смешнее"/"убери X"/"добавь Y" — **правь последний показанный вариант прямо в чате**, не создавай новый скилл-вызов.

## Сохранение

Когда юзер выбрал номер:

1. Создай `контент/посты/черновики/` если её нет (проактивно предложи структуру: "Разложу по идеи/черновики/опубликовано?" если это первый пост)
2. Если директории по лайфцикл нет — спроси: "Разложу по черновики/опубликовано/ или один в контент/посты/?"
3. Имя файла: `YYYY-MM-DD-краткая-тема.md`
4. Фронтматтер:

```yaml
---
type: пост
created: YYYY-MM-DD
format: [информационный | сторителлинг | продающий]
topic: $ARGUMENTS
status: черновик
sources:
  - профиль/стиль.md
  - профиль/бизнес.md
---
```

5. Трассировка в конце:
```markdown
<!-- Трассировка -->
Основано на:
- Стиль: профиль/стиль.md
- Бизнес: профиль/бизнес.md
- Персона: [если применялась]
- Правила: [применённые]
- История: [N последних постов]
```

6. Подтверждение:
```
Сохранено: контент/посты/черновики/YYYY-MM-DD-тема.md

Опубликуешь — скажи "опубликовал", перенесу в опубликовано/.
```

## После публикации

Когда юзер говорит "опубликовал"/"выложил"/"запостил" — переложи последний созданный пост из `черновики/` в `опубликовано/` и обнови `status: опубликован` во фронтматтере.
