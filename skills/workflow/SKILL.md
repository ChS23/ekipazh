# Интеграция инструмента

## Когда включается

- Пользователь рассказывает "я использую Telegram для...", "веду Instagram", "отвечаю клиентам в WhatsApp"
- Просит "помоги настроить под мой Telegram", "хочу автоматизировать ответы"
- Описывает рутинный процесс с инструментом

## Что делать

### 1. Узнай контекст

Прочитай `профиль/бизнес.md` и `профиль/стиль.md` для контекста.
Прочитай существующие правила в `правила/` — не дублируй.

### 2. Проведи интервью

```
Расскажи, как ты используешь [инструмент]:
1. Для чего? (продажи, контент, поддержка, учёт)
2. Как часто? (каждый день, 3 раза в неделю, по запросу)
3. Какой типичный процесс? (получил запрос → ответил → ...)
4. Что бесит / на что уходит время?
```

## Отслеживание неопределённости

### Критические разделы (70% веса) — 5 пунктов:
1. Для чего использует инструмент (цель)
2. Триггер (что запускает процесс)
3. Шаги (что делает по порядку)
4. Формат результата (текст, файл, длина)
5. Чего НЕ делает (антипаттерны)

### Вспомогательные разделы (30% веса) — 3 пункта:
1. Как часто (ритм)
2. Ограничения платформы (длина, формат, специфика)
3. Пример хорошего результата

### Расчёт

```
Неопределённость = (незаполненных_крит / 5) × 0.7 + (незаполненных_вспом / 3) × 0.3
Цель: ≤ 0.1
```

После каждого ответа пользователя выводи: `[Неопределённость: 0.XX]`

Уточняй по Clarification Loop пока неопределённость > 0.1.

### 3. Создай правило

На основе интервью создай правило в `правила/`:

```yaml
---
type: правило
created: YYYY-MM-DD
applies-to: [тип — определи по контексту]
sources:
  - профиль/стиль.md
---
```

Правило фиксирует: когда → что делать → чего не делать → хороший результат.

### 4. Создай персональный скилл

Создай скилл в `скиллы/` (директория в рабочей папке пользователя, не в плагине):

```
скиллы/
  тг-пост/
    SKILL.md
```

Скилл должен:
- Иметь понятное описание для автоматического вызова
- Читать профиль и правила
- Учитывать формат инструмента (длина, стиль, ограничения платформы)
- Генерировать результат готовый к использованию

**Пример SKILL.md для Telegram-поста:**
```markdown
---
name: tg-post
description: Создать пост для Telegram-канала. Используй когда пользователь просит пост для ТГ, или говорит "напиши в канал".
allowed-tools: Read, Write, Glob, Grep
argument-hint: [тема]
---

# Пост для Telegram-канала

## Контекст
1. Прочитай профиль/стиль.md
2. Прочитай правила/ — все применимые
3. Прочитай профиль/бизнес.md

## Формат Telegram
- До 1000 символов (оптимально 500-800)
- Первая строка — крючок (без кликбейта)
- Абзацы по 2-3 предложения с пустой строкой между
- Эмодзи: максимум 2-3, не в каждой строке
- CTA в конце
- Хэштеги: 3-5 в конце, если уместно

## Генерация
Создай 2 варианта на тему $ARGUMENTS
Покажи оба, спроси какой ближе.
```

### 5. Проверь на реальном примере

```
Скилл создан. Давай проверим — дай мне реальную задачу для [инструмент].
```

Сгенерируй результат через новый скилл. Спроси: "Похоже? Что не так?"
Если не похоже — поправь правило и скилл.

### 6. Итог

```
Готово ✓

Создано:
  правила/правило-NNN-[инструмент].md
  скиллы/[название]/SKILL.md

Теперь можешь просто написать "[задача] для [инструмент]" — Экипаж сделает по твоим правилам.
Хочешь настроить ещё один инструмент?
```

## Типичные интеграции

| Инструмент | Что создаём |
|---|---|
| Telegram-канал | Скилл /тг-пост (короткий формат, эмодзи, хэштеги) |
| Instagram | Скилл /инста-пост (визуал-описание + текст + хэштеги) |
| WhatsApp | Скилл /ответ-wa (короткие ответы клиентам) |
| Email-рассылка | Скилл /рассылка (тема + текст + CTA) |
| Авито/маркетплейс | Скилл /объявление (заголовок + описание + характеристики) |
| Google Таблицы | Правило для формата данных |

## Важно

- Скилл создаётся в рабочей директории пользователя (`скиллы/`), НЕ в плагине
- Каждый скилл — самодостаточный, читает профиль и правила
- Не перегружай: один скилл = один инструмент + один процесс
- Формат платформы важнее: пост в ТГ ≠ пост в Instagram ≠ пост в ВК
