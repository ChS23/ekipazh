# Рынок и аудитория

ultrathink


## Твоя роль

Ты — аналитик рынка (см. `agents/analyst/AGENT.md`). Твоя задача — провести исследование и создать 4 артефакта.

## Предварительные проверки

1. Проверь наличие `профиль/бизнес.md`
2. Если файла нет — ответь:
   ```
   Для анализа рынка нужен контекст бизнеса.
   Запусти /start [название-бизнеса] — это займёт 5-10 минут.
   ```
   И **останови выполнение**.
3. Если файл есть — прочитай `профиль/бизнес.md` и `профиль/стиль.md`

## Начало

```
Начинаю анализ рынка для [бизнес из профиля].

На основе профиля у меня уже есть базовое понимание.
Сейчас уточню несколько моментов, чтобы анализ был точным.
```

## Отслеживание неопределённости

### Критические разделы (70% веса) — 5 пунктов:
1. Кто идеальный клиент (конкретный человек, не группа)
2. Какая у него боль (что мешает прямо сейчас)
3. Кто основные конкуренты (3-5)
4. Чем отличаемся от конкурентов
5. Кто точно НЕ наш клиент (анти-персона)

### Вспомогательные разделы (30% веса) — 5 пунктов:
1. Размер рынка (порядок)
2. Тренды (что меняется)
3. Сезонность
4. Типичные возражения клиентов
5. Триггеры покупки

### Расчёт

```
Неопределённость = (незаполненных_крит / 5) × 0.7 + (незаполненных_вспом / 5) × 0.3
Цель: ≤ 0.1
```

После каждого ответа пользователя выводи: `[Неопределённость: 0.XX]`

## Clarification Loop

### Фаза 0: Загрузка контекста

1. Прочитай `профиль/бизнес.md` — извлеки уже известное о клиентах, рынке
2. Прочитай `профиль/стиль.md` — тон, ценности
3. Если есть `профиль/состояние.md` — извлеки контекст
4. Заполни то, что уже знаем — не задавай вопросы повторно

### Фаза 1: Анализ

После каждого ответа пользователя:
1. Извлеки информацию
2. Заполни пункты
3. Пересчитай неопределённость

### Фаза 2: Вопросы

Пока неопределённость > 0.1:
1. Задай 2-3 вопроса по приоритету
2. Начинай с персон — это самое важное

**Примеры хороших вопросов:**
- "Кто твой идеальный клиент? Опиши конкретного человека, а не группу."
- "Когда клиент выбирает между тобой и конкурентом — что решает?"
- "Кого из конкурентов уважаешь? Что они делают хорошо?"
- "Есть ли сезонность? Когда пик, когда затишье?"
- "Какие возражения слышишь чаще всего?"
- "А кто точно НЕ твой клиент? Кому ты бы отказал?"
- "Кто покупает первым — кто не боится попробовать?"

### Фаза 3: Обработка "Не знаю"

1. **Предложи гипотезу:**
   - "На основе профиля, предполагаю что твой рынок — [X]. Это близко?"

2. **Предложи варианты:**
   - "Обычно в [отрасли] конкурируют по: (а) цене, (б) качеству, (в) сервису. Как у тебя?"

3. **Пометь как гипотезу:**
   - "[гипотеза] Основной конкурент — [X], на основании [источник]"

### Фаза 4: Завершение

Когда неопределённость ≤ 0.1:

1. Скажи: "Достаточно информации. Формирую анализ."
2. Сгенерируй 4 артефакта
3. Покажи сводку ключевых инсайтов
4. Спроси: "Что думаешь? Совпадает с ощущением рынка?"

### Фаза 5: Финализация

1. Внеси правки
2. Создай директорию `анализ/` и `анализ/персоны/` (если нет)
3. Сохрани файлы:

**анализ/рынок.md:**
```yaml
---
type: рынок
created: YYYY-MM-DD
sources:
  - профиль/бизнес.md
---
```

Содержимое:
```
# Карта рынка

## Рынок
...

## Объём
...

## Тренды
...

## Сезонность
...

## Ключевой инсайт
...
```

**анализ/персоны/ (по файлу на персону):**

Типы персон:
- **Основная** — главный клиент, для кого всё строится
- **Early Adopter** — кто купит первым, готов терпеть недоделки
- **Анти-персона** — кто точно НЕ наш клиент (помогает фокусироваться)

Используй шаблон из `skills/analyze/templates/persona.md`.
Файлы: `анализ/персоны/персона-имя.md`, `анализ/персоны/анти-персона.md`

**анализ/конкуренты.md:**
```yaml
---
type: конкуренты
created: YYYY-MM-DD
sources:
  - профиль/бизнес.md
---
```

Содержимое:
```
# Конкуренты

| Конкурент | Сильные стороны | Слабые стороны | Цена | Позиционирование |
|-----------|-----------------|----------------|------|------------------|
| ...       | ...             | ...            | ...  | ...              |

## Наше отличие
...

## Ключевой инсайт
...
```

**анализ/swot.md:**
```yaml
---
type: swot
created: YYYY-MM-DD
sources:
  - профиль/бизнес.md
  - профиль/стиль.md
  - анализ/рынок.md
  - анализ/конкуренты.md
---
```

Содержимое:
```
# SWOT-анализ

## Сильные стороны
- ...

## Слабые стороны
- ...

## Возможности
- ...

## Угрозы
- ...

## Ключевой инсайт
...

## Рекомендации
...
```

4. Создай решение (если выявлен ключевой выбор — целевой сегмент, фокус и т.п.)
5. Выведи итог:

```
Анализ готов ✓

  анализ/рынок.md
  анализ/персоны/персона-[имя].md (×N)

Ключевые инсайты:
  1. ...
  2. ...
  3. ...

Следующий шаг: /post и /kp теперь учитывают персоны и конкурентное окружение.
```
