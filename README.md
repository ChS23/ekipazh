# Экипаж

Плагин для Claude Code. Превращает его в ИИ-команду для эксперта или самозанятого.

Работаешь один — сам продюсер, маркетолог, копирайтер. Экипаж помогает с рутиной: пишет посты в твоём стиле, готовит КП, обрабатывает возражения. По твоим правилам, не по своим.

## Установка

```bash
git clone https://github.com/ChS23/ekipazh.git
claude --plugin-dir ./ekipazh/
```

Затем в сессии:
1. `/config` → Output style → **Экипаж**
2. Перезапусти сессию

## Разрешения (чтобы не спрашивало каждый раз)

По умолчанию Claude Code спрашивает подтверждение на каждую операцию. Чтобы работал без трения, добавь в свой `~/.claude/settings.json`:

```json
{
  "permissions": {
    "allow": [
      "Read",
      "Write",
      "Glob",
      "Grep",
      "Bash(typst *)",
      "WebSearch",
      "WebFetch"
    ]
  },
  "defaultMode": "acceptEdits"
}
```

- `Read`, `Write`, `Glob`, `Grep` — файловые операции, работают одинаково на Linux/Mac/Windows
- `Bash(typst *)` — для генерации PDF через `/format`
- `WebSearch`, `WebFetch` — для `/analyze` (поиск данных о рынке)
- `defaultMode: "acceptEdits"` — автоматически принимает правки файлов без подтверждения

## Что есть

| Скилл | Что делает |
|-------|------------|
| `/start [бизнес]` | Выгружает бизнес в систему — профиль, стиль, состояние |
| `/analyze` | Рынок, персоны, анти-персона, конкуренты, SWOT |
| `/rule [задача]` | Формализует опыт в правило. Правила применяются при генерации |
| `/post [тема]` | Пост для соцсетей в стиле бренда — 3 варианта |
| `/kp [клиент]` | КП с 2 тарифами под конкретного клиента |
| `/objection [суть]` | Разбор возражения + 2 варианта ответа |
| `/format` | Оформить артефакт в PDF через Typst |
| `/workflow [инструмент]` | Создать правило и персональный скилл под инструмент |
| `/status` | Что настроено, что нет |

Скиллы не нужно вызывать вручную — Экипаж сам включает нужный по контексту.

## Как работает

```
Бизнес  →  Артефакты (профиль, стиль, персоны)
Опыт    →  Правила (как делать задачи)
Артефакты + Правила  →  Контент (посты, КП, возражения)
```

Ты управляешь. Экипаж исполняет. ИИ не заменяет — ускоряет.

## Структура артефактов

После работы в твоей рабочей папке:

```
профиль/     — бизнес.md, стиль.md, состояние.md
анализ/      — рынок.md, персоны/, конкуренты.md, swot.md
правила/     — правило-NNN-*.md
контент/     — посты/, кп/, возражения/
решения/     — решение-NNN-*.md
```

Всё в Markdown, версионируется, переносится. Хоть в git храни.

## Формула неопределённости

В скиллах сбора информации (`/start`, `/analyze`, `/rule`, `/kp`, `/objection`, `/workflow`) система выводит `[Неопределённость: 0.XX]` — насколько полно собрана информация. Цель ≤ 0.1. Так понятно когда хватит вопросов.

## Для разработки

Перезагрузка без рестарта:
```
/reload-plugins
```

Отладка:
```bash
claude --debug --plugin-dir ./ekipazh/
```

├── agents/                       — 3 субагента (interviewer, analyst, operator)
├── hooks/hooks.json              — SessionStart напоминание
├── scripts/                      — bash + powershell
